**项目架构、遇到问题、解决方案、重新设计怎么做**

1.假设后端从Spring Boot 2.x升级到Spring Boot 3.x 以及JDK从8升级到17。你会关注哪些哪些问题？

​	**考察意图：**对spring版本和JDK版本的理解，以及对整体架构版本的理解、把控。
​    **理想答案：** 至少要知道javax包变成jakarta，关心兼容性，知道各版本的一些差异。

2.每日打卡。没人每天只能打卡一次。如何保证幂等性？（ **淘汰回答：**回答简单，只加锁或只前端处理。）

​	**考察意图：**分布式锁 + 数据库唯一索引 + 业务幂等设计。
​    **理想答案：** 至少要redis的 setnx锁或数据库**用户+时间**唯一索引。

3.反馈使用过程中有Bug，怎么检查？（**淘汰回答：** 推诿或不主动、及时处理、找借口甩锅。）

​	**考察意图：** Bug排查的标准化流程（而非瞎蒙）。
​	**理想答案：** 复现、看日志、看接口、定位、Arthas在线定位。

4.反馈，APP用起来很卡，怎么检查？可能存在哪些问题？（**淘汰回答：** 直接说“重启服务器就好了”。）

​	**考察意图：** 性能排查思路（全链路）。
​	**理想答案：** 网络、后端、服务器（cpu、内存、带宽）、序列/反序列化、数据库连接池、Full GC。

5.展示的列表信息，会包含很多出了列表数据的额外信息，比如用户、点赞、收藏、评论、黑名单、不喜欢等。你的实现思路。（**淘汰回答：** 直接数据库一个SQL全部查询）

​	**考察意图：**  **大数据量下的“连表查询 vs 冗余设计 vs 聚合”**，特别重要。
​	**理想答案：** 主数据按分页、关系数据去重查询、聚合数据提前落库/redis。计数数据冗余存储。

6.阿里云体系了解程度、服务器了解程度、微信体系了解程度。（ **淘汰回答：**都不了解、都没做过）

​	**考察意图：**是否实际使用过阿里云、微信。是否操作过服务器。
​    **理想答案：**都对接、使用过。至少需要熟悉linux简单操作。

7.开发过程中，领导有新的需求进来，要求尽快处理，怎么解决？（**淘汰回答：**直接说不）

​    **考察意图：优先级管理与向上沟通**（软技能）
​    **理想答案：**不拒绝、评估工作量、向上汇报、重新排版重要性、提出解决或临时应急方案。

8.非工作时间，公司群有提出Bug，怎么应急处理（非频繁）（**淘汰回答：**不评优先级上班再说）

​	**考察意图：**值班意识**和**事故响应应对。（应急情况应对）
​	**理想答案：**快速响应、评估严重性、根据严重性及时或延时处理，事后复盘。

9.如果给你一个方向或演示包，如何落地成可用项目，如何设计整个系统？有哪些考量因素？

​	**考察意图：**从0到1的架构设计能力**（极为重要）****淘汰回答：**直接用IDE运行Demo，说“能跑就行”。
​	**理想答案：**尽量向上获取更多信息、评估、选型、设计方案、多AI设计。架构要根据实际业务，不是哪个高级用哪个。

10.现在使用什么AI、哪些AI工具。平常获取AI资讯的渠道。AI代码在开发中的占比，用什么方式提升效率。AI写的代码如何保证质量？怎么处理AI代码产生的幻觉？（**淘汰回答：**有人提醒才去看相关内容。）

​	**考察意图：**工程效率意识**（5年经验应当善用工具）。**
​	**理想答案：**Github、各种社区、各种群组、公众号获取知识，AI生成单元测试、接口文档压力测试。
