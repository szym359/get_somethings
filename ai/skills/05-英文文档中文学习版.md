## Build skills 中文学习版

`Build skills` 主要讲如何创建和使用 skill。

你可以把它理解成：

> 把一类可复用的任务流程、知识、规则、模板或脚本，整理成 Codex 可以在需要时加载的能力包。

核心知识点：

- skill 是一个目录
- 每个 skill 至少包含 `SKILL.md`
- `SKILL.md` 需要 frontmatter
- frontmatter 至少包含 `name` 和 `description`
- skill 可以被显式调用
- skill 也可以根据 `description` 隐式匹配
- 项目级 skill 可以放在 `.agents/skills`
- 大段资料可以放进 `references/`
- 可执行逻辑可以放进 `scripts/`
- 模板和素材可以放进 `assets/`

一个最小 skill：

```text
.agents/skills/customer-reply/SKILL.md
```

```markdown
---
name: customer-reply
description: 用于客服回复、投诉处理、退款沟通和售后安抚。
---

# Customer Reply

- 先识别用户诉求
- 语气礼貌、克制、具体
- 不承诺超出政策的补偿
- 给出下一步可执行方案
```

## Custom instructions with AGENTS.md 中文学习版

`Custom instructions with AGENTS.md` 主要讲如何给 Codex 设置项目级长期说明。

你可以把它理解成：

> `AGENTS.md` 是写给 Codex 的项目说明书。它告诉 Codex 在这个项目里工作时应该遵守哪些默认规则。

适合写进去的内容：

- 项目结构说明
- 安装依赖命令
- 测试命令
- 构建命令
- 代码风格要求
- 不要修改的文件
- 提交或 PR 规范
- 特定目录的注意事项

一个简单例子：

```markdown
# Project Instructions

- 使用中文回答用户。
- 修改前先阅读相关模块已有风格。
- 后端代码在 `backend/`。
- 提交前运行项目测试。
- 不要修改自动生成文件。
```

## 常见英文术语对照

| 英文 | 中文理解 |
| --- | --- |
| skill | 技能、专用能力包 |
| `SKILL.md` | skill 的入口说明文件 |
| frontmatter | Markdown 文件开头的元信息区 |
| `name` | skill 的真实名称 |
| `description` | skill 的用途描述和触发说明 |
| explicit invocation | 显式触发，直接点名使用 |
| implicit invocation | 隐式触发，Codex 根据描述自动判断 |
| references | 引用资料、规则、案例 |
| scripts | 可执行脚本 |
| assets | 模板、图片、素材 |
| `AGENTS.md` | 项目级 Codex 默认说明文件 |
| plugin | 插件，通常比 skill 更完整，可包含工具、MCP、资产等 |
| MCP | 让 Codex 连接外部工具或数据源的协议/机制 |

## 学习建议

不要一开始就追求完整结构。

推荐路线：

1. 先写一个最小 skill
2. 用 `$skill-name` 显式触发
3. 等规则变多，再拆 `references/`
4. 有重复机械动作时，再加 `scripts/`
5. 有模板或素材时，再加 `assets/`
6. 项目通用规则再放进 `AGENTS.md`
